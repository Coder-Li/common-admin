# User Session Management Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement administrator-facing online session and login device management for internal Common Admin users.

**Architecture:** Reuse the existing Prisma `UserSession` table as the server-side source of truth. Add a focused NestJS `UserSessionModule` for admin listing and single-session revocation, then expose the contract through OpenAPI/Orval and add a permission-aware admin table page at `/session-management`.

**Tech Stack:** NestJS 11, Prisma, PostgreSQL, class-validator, Swagger, Jest, React 19, Vite, TypeScript, TanStack Query, TanStack Router, Orval, Vitest, React Testing Library, Zustand.

## Implementation Notes

- Do not modify or commit `docs/next-step.md`.
- Do not add third-party login fields.
- Do not add a Prisma migration or a new device table.
- Do not add Redis, token blacklists, batch revocation, trusted devices, device fingerprints, or "revoke all user sessions".
- Use an internal user-agent summary helper for v1. Do not add a UA parser dependency unless implementation proves the helper is insufficient.
- The backend must remain authoritative for permission checks and revocation.
- The frontend must use generated OpenAPI/Orval endpoint functions and schema types.
- Keep revocation and audit-log insert in the same Prisma transaction.
