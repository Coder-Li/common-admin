# Admin I18n Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Simplified Chinese and English UI copy support to the admin frontend.

**Architecture:** Build a small local React i18n layer with typed dictionaries, locale resolution/persistence helpers, a Provider/hook, and a shared language switcher. Existing views will read user-facing copy through `useI18n()` while keeping `Common Admin` untranslated.

**Tech Stack:** Vite, React, TypeScript, Vitest, React Testing Library, localStorage.

---

## File Structure

- Create `apps/admin/src/i18n/messages.ts`: supported locale constants, dictionary, translation keys, interpolation values.
- Create `apps/admin/src/i18n/locale-storage.ts`: browser/storage helpers for resolving and persisting locale.
- Create `apps/admin/src/i18n/I18nProvider.tsx`: React context, `useI18n()`, translator function, provider state.
- Create `apps/admin/src/i18n/LanguageSwitcher.tsx`: shared `中文 / EN` switcher.
- Create `apps/admin/src/i18n/locale-storage.test.ts`: locale resolution and persistence tests.
- Create `apps/admin/src/features/auth/LoginView.test.tsx`: login copy and switch persistence component tests.
- Create `apps/admin/src/layouts/AdminShell.test.tsx`: authenticated shell copy in Chinese/English.
- Modify `apps/admin/src/App.tsx`: wrap app content in `I18nProvider`.
- Modify `apps/admin/src/features/auth/LoginView.tsx`: replace hard-coded user-facing copy and auth toasts.
- Modify `apps/admin/src/layouts/AdminShell.tsx`: replace hard-coded shell copy and add switcher.
- Modify `apps/admin/src/pages/DashboardContent.tsx`: replace hard-coded dashboard copy.
