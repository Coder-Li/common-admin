# Next Step

这是一份临时路线图，按 Common Admin 作为管理后台启动项目的复用价值来排优先级。

当前项目已具备的底座：

- 认证与会话
- RBAC 权限
- 用户 / 角色 / 权限
- 字典
- 文件管理
- 审计日志
- 系统设置
- OpenAPI 生成
- 日志与质量门禁

后续建议按下面顺序单独拆功能实现。

## P0 必补

### 1. 在线会话 / 登录设备管理

建议内容：

- 会话列表
- 登录时间、IP、UA
- 过期时间
- 会话撤销 / 强制下线
- 最近登录记录

价值：补齐认证闭环，安全感最直接。

### 2. 组织 / 部门 / 岗位

建议内容：

- 部门树
- 岗位列表
- 用户归属部门 / 岗位
- 基础组织关系维护

价值：多数企业后台迟早会用到。

### 3. 数据权限框架

建议内容：

- 全部
- 本人
- 本部门
- 本部门及子部门
- 自定义部门

价值：避免后续每个业务模块各写各的数据过滤逻辑。

### 4. 菜单 / 路由配置化

建议内容：

- 菜单名称
- 路径
- 图标
- 排序
- 是否隐藏
- 绑定权限码

价值：让后台更像可复用底座，而不只是静态前端壳子。

## P1 很值得做

### 5. 登录日志

建议内容：

- 登录成功 / 失败
- 失败原因
- IP、UA
- 登录用户

### 6. 通知 / 公告中心

建议内容：

- 公告管理
- 站内通知
- 已读状态

### 7. 导入 / 导出

建议内容：

- CSV / XLSX 导出
- 模板下载
- 行级失败回显
- 常用基础模块导入

### 8. 批量操作

建议内容：

- 批量删除
- 批量启停
- 批量分配角色

### 9. 任务中心

建议内容：

- 任务状态
- 进度
- 错误信息
- 操作人
- 开始 / 结束时间

### 10. 系统设置增强

建议内容：

- 邮件设置
- 短信设置
- 登录安全策略
- 密码策略
- 站点开关

## P2 按定位再做

### 11. 多租户

适合平台型产品或多客户复用场景。

### 12. SSO / 第三方登录

适合企业内部系统集成。

### 13. 工作流 / 审批流

适合明确要做流程平台时再上。

### 14. 报表看板配置器

适合真实指标和运营场景稳定后再抽象。

### 15. 文档站

适合把架构、模式、AI 开发协议、部署文档统一发布出去。

## 推荐执行顺序

1. 在线会话管理
2. 登录日志
3. 组织 / 部门 / 岗位
4. 数据权限框架
5. 菜单 / 路由配置化
6. 导入 / 导出
7. 批量操作
8. 任务中心
9. 通知 / 公告中心
10. 多租户或 SSO，按业务定位决定

## 备注

这份文档只作为临时路线图使用。后面每个功能点建议单独开一个实现任务、单独写设计说明、单独做验证。
